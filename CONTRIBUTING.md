# 🤝 Contributing to OpenBox

## Development Process
1. Fork the repository
2. Create a feature branch
3. Make your changes
4. Run tests
5. Submit a pull request

## Code Style
- Shell: ShellCheck compliant
- Python: PEP 8
- YAML: Valid syntax
- Documentation: Markdown

## Security Guidelines
1. Never commit secrets
2. Use 2FA for all accounts
3. Sign commits with GPG
4. Follow security best practices

## Pull Request Process
1. Update documentation
2. Add tests
3. Ensure CI passes
4. Get code review
5. Merge with squash

## Branch Naming
- `feature/*` - New features
- `fix/*` - Bug fixes
- `security/*` - Security fixes
- `docs/*` - Documentation
- `chore/*` - Maintenance
